framework.task¶

task¶

Task¶

class ding.framework.task.Task(async_mode: bool = False, n_async_workers: int = 3, middleware: Optional[List[Callable]] = None, step_wrappers: Optional[List[Callable]] = None, labels: Optional[Set[str]] = None, **_)[source]¶

Tash will manage the execution order of the entire pipeline, register new middleware, and generate new context objects.

__init__(async_mode: bool = False, n_async_workers: int = 3, middleware: Optional[List[Callable]] = None, step_wrappers: Optional[List[Callable]] = None, labels: Optional[Set[str]] = None, **_) → None[source]¶
backward(backward_stack: List[Generator] = None) → ding.framework.task.Task[source]¶
Overview:

Execute the rest part of middleware, by the reversed order of registry.

emit(event: str, *args, only_remote: bool = False, only_local: bool = False, **kwargs) → None[source]¶
Overview:

Emit an event, call listeners.

Arguments:

  • event (str): Event name.

  • only_remote (bool): Only broadcast the event to the connected nodes, default is False.

  • only_local (bool): Only emit local event, default is False.

  • args (any): Rest arguments for listeners.

forward(fn: Callable, ctx: ding.framework.context.Context = None, backward_stack: List[Generator] = None) → ding.framework.task.Task[source]¶
Overview:

This function will execute the middleware until the first yield statment, or the end of the middleware.

Arguments:

  • fn (Callable): Function with contain the ctx argument in middleware.

on(event: str, fn: Callable) → None[source]¶
Overview:

Subscribe to an event, execute this function every time the event is emitted.

Arguments:

  • event (str): Event name.

  • fn (Callable): The function.

once(event: str, fn: Callable) → None[source]¶
Overview:

Subscribe to an event, execute this function only once when the event is emitted.

Arguments:

  • event (str): Event name.

  • fn (Callable): The function.

renew() → ding.framework.task.Task[source]¶
Overview:

Renew the context instance, this function should be called after backward in the end of iteration.

run(max_step: int = 10000000000) → None[source]¶
Overview:

Execute the iterations, when reach the max_step or task.finish is true, The loop will be break.

Arguments:

  • max_step (int): Max step of iterations.

sequence(*fns: List[Callable]) → Callable[source]¶
Overview:

Wrap functions and keep them run in sequence, Usually in order to avoid the confusion of dependencies in async mode.

Arguments:

  • fn (Callable): Chain a sequence of middleware, wrap them into one middleware function.

stop() → None[source]¶
Overview:

Stop and cleanup every thing in the runtime of task.

use(fn: Callable, filter_labels: Optional[Iterable[str]] = None) → ding.framework.task.Task[source]¶
Overview:

Register middleware to task. The middleware will be executed by it’s registry order.

Arguments:

  • fn (Callable): A middleware is a function with only one argument: ctx.

use_step_wrapper(fn: Callable) → ding.framework.task.Task[source]¶
Overview:

Register wrappers to task. A wrapper works like a decorator, but task will apply this decorator on top of each middleware.

Arguments:

  • fn (Callable): A wrapper is a decorator, so the first argument is a callable function.
